<objective>
Wire the toggle (Plan 02-01) to the payload (Plan 02-02). When the user clicks an exemplar card while `exemplarMode === 'demo'`, the app:
1. Looks up the bundled JSON via `getDemoPayload(slug)`,
2. Validates it via the existing `validateSession()` gate,
3. Hydrates input + mapping stores via the same code path as Stage 7A's "load session file" feature,
4. If `pipeline_version` or `folio_version` in the payload disagrees with the runtime, surfaces a non-blocking, dismissible warning banner.

Purpose: Deliver the actual demo experience and prove the spike-001 invariant — that a demo payload round-trips through the same load path as a user-saved session.

Output: Refactored `useSession` (extracted `loadSessionFromObject`), updated `handleExemplarSelect` in `App.tsx`, a new `StalePresetBanner` component, and an extended `demo-store` for warning state.
</objective>
