    Algorithm:
      1. Read `scripts/demos/{area}.input.json`. Fail fast if missing.
      2. Resolve LLM API key from env: `ANTHROPIC_API_KEY` for `--provider anthropic`, `OPENAI_API_KEY` for `openai`, `GOOGLE_API_KEY` for `google`. Fail fast with a clear error if absent.
      3. Parse the tabbed text into `ParseItem[]` (mirror what the backend `/api/parse/text` does — easiest is to POST `text` to `<backend>/api/parse/text` and consume the returned `parse_result.items`). Capture both `parse_result` (full) and `items` for the pipeline call.
      4. POST `/api/pipeline/map` with: `{ items, llm_config: {provider, model, api_key: null, base_url: null}, threshold, max_per_branch, mandatory_branches: [] }`. Send the API key via header `X-LLM-API-Key` (or whatever convention `extract_api_key` in `backend/app/services/auth.py` honors — verify by reading it). Set header `X-Local-Token` if `FOLIO_MAPPER_NO_AUTH` is not set in the environment of the backend.
      5. Receive `PipelineResponse` (`mapping` + `pipeline_metadata`).
      6. Snapshot versions:
         - `folio_version`: `from folio import FOLIO; folio = FOLIO(); folio_version = getattr(folio, 'owl_version', None) or _git_short_sha_of_folio_python_install()`. Fall back to `'unknown'` only if both fail.
         - `pipeline_version`: read from `apps/desktop/package.json` `version` field (this is the source-of-truth version per project MEMORY) AND attempt `git rev-parse --short HEAD` for a commit suffix; combine as `f"{pkg_version}+{git_sha}"`.
      7. Auto-accept the top-scored candidate per item: build `selections: Record<int, string[]>` where each key is the item index and the value is `[top_candidate_iri_hash]` derived from `mapping.results[idx].candidates[0].iri_hash` when `candidates` is non-empty AND the candidate's `score` ≥ `threshold * 100`. Items below threshold get `[]` (left for human curation in the demo).
      8. Build `node_statuses`: `'completed'` for items with non-empty selections, `'unmapped'` otherwise.
      9. Construct the SessionFile dict per the schema in `packages/core/src/session/index.ts`:
         - `version`: `"1.3"` (mirror `SESSION_VERSION`)
         - `created` / `updated`: ISO 8601 UTC now
         - `source_file`: `null`
         - `input_format`: `parse_result.format` (`'hierarchical'` for PI)
         - `total_nodes`: `len(items)`
         - `completed`: count of `node_statuses == 'completed'`
         - `skipped`: 0
         - `current_position`: 0
         - `provider`: the `--provider` arg
         - `model`: the resolved model id
         - `text_input`: the raw text from the input JSON
         - `parse_result`: from step 3
         - `mapping_response`: `pipeline_response.mapping`
         - `pipeline_metadata`: `pipeline_response.pipeline_metadata`
         - `selections`, `node_statuses`: from steps 7-8
         - `notes`: {}
         - `screen`: `"mapping"`
         - `branch_states`, `input_branch_states`: {}
         - `branch_sort_mode`: `"default"`
         - `custom_branch_order`: []
         - `status_filter`: `"all"`
         - `suggestion_queue`, `review_queue`: []
      10. Add the two NON-schema snapshot fields at the top level: `pipeline_version` and `folio_version`. They are extra keys; `validateSession()` in core ignores unknown keys (it is a structural shape-check, not a schema-strict validator — confirmed by reading the function), so this is safe.
      11. Write JSON pretty-printed (2-space indent) to `apps/web/src/exemplar/demos/{slug}.demo.json`.
      12. Print a summary to stderr: items count, completed-vs-unmapped, fan-out histogram (1:1, 1:2, 1:3, 1:4+), pipeline_version, folio_version, output path.
