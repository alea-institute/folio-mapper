    export function detectStalePreset(args: {
      payloadPipelineVersion: string | null;
      payloadFolioVersion: string | null;
      runtimePipelineVersion: string | null;
      runtimeFolioVersion: string | null;
    }): null | { /* same shape as args */ } {

      return stale ? args : null;
    }

    ## Curating a Demo Payload
    Operator-facing workflow for regenerating `apps/web/src/exemplar/demos/{slug}.demo.json` after a material pipeline or FOLIO change.

    ### When to recurate
    - FOLIO ontology version bumped and IRIs shifted
    - Pipeline scoring logic changed (Stage 2/3 LLM behavior, threshold defaults)
    - New practice area being added (mirror spike 001 first to validate FOLIO coverage; see `.planning/spikes/001-demo-pi-curation/README.md`)

    ### Prerequisites
    - Working LLM API key for one of: Anthropic, OpenAI, Google
    - `backend/.venv` with `folio-python` and the project's other backend deps installed (`pnpm install:backend` or equivalent)
    - The frontend backend running locally on port 58000 (`pnpm dev:api`)

    ### Workflow
    ```bash
    # Terminal A
    pnpm dev:api

    # Terminal B
    export ANTHROPIC_API_KEY=sk-ant-...        # or OPENAI_API_KEY / GOOGLE_API_KEY
    backend/.venv/bin/python scripts/curate_demos.py \
      --area personal-injury \
      --provider anthropic
    # → writes apps/web/src/exemplar/demos/personal-injury.demo.json

    # Sanity-check the output
    jq '.version, .total_nodes, .pipeline_version, .folio_version' \
       apps/web/src/exemplar/demos/personal-injury.demo.json

    # Commit
    git add apps/web/src/exemplar/demos/personal-injury.demo.json
    git commit -m "feat(demos): regenerate personal-injury after pipeline 0.x.y"
    ```

    ### Adding a new practice area
    1. Run a coverage probe: copy `.planning/spikes/001-demo-pi-curation/run_coverage_probe.py` and adapt the items list. Goal: confirm at least 3 enrichment candidates with ≥4 high-confidence FOLIO matches each.
    2. Author `scripts/demos/{slug}.input.json` with the lean exemplar text + 2-3 enrichment items in their thematic branches.
    3. Add the slug to `apps/web/src/exemplar/demos/index.ts` `DEMO_PAYLOADS` map (next to `personal-injury`).
    4. Run the curation script per the workflow above.
    5. Run `pnpm vitest run` to confirm round-trip + no-network tests still pass.

    ### What NOT to commit
    - LLM API keys (the script never embeds them, but verify with `grep -E "(sk-|api_key.*[a-zA-Z0-9]{20,})" apps/web/src/exemplar/demos/{slug}.demo.json` before committing).
    - PII or client matter content — every demo payload is shipped to every user; treat it as public.

    ### See also
    - `.planning/phases/02-demo-mode/` — phase plans + context
    - `.planning/spikes/001-demo-pi-curation/README.md` — feasibility findings + enrichment selection rationale
