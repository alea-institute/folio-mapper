<objective>
Add a "Rename" control to each row of the session picker so users can give a session a human-friendly name, replacing the auto-derived "Untitled"/"Pasted text" label. The custom name is stored on the session registry record, displayed in the picker (and anywhere the session label is shown), and persists across reloads.

Purpose: Sessions are currently labeled only by their auto-derived source ("Untitled" for pasted text with no filename), making multiple saved sessions hard to tell apart in the picker.

Output: A new `customName` field + `renameSession()` function in the registry, onWrite preservation of the custom name, an accessible inline edit-in-place rename control in the picker, and App.tsx wiring that updates the registry and refreshes the picker list immediately. Tests covering the registry rename + onWrite-preservation path and the picker render.
</objective>
